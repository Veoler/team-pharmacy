# Бекенд для приложения онлайн-аптеки

REST API для онлайн-аптеки, реализованный на языке Go. Проект разработан в соответствии с принципами чистой слоистой архитектуры, использует строгую валидацию бизнес-правил и предоставляет предсказуемый интерфейс для фронтенд-команды.

## 🛠️ Технологический стек

*   **Язык разработки:** Go (Golang)
*   **Веб-фреймворк:** Gin Gonic
*   **База данных:** PostgreSQL

## 📐 Архитектура проекта

Кодовая база строго разделена на изолированные слои с четкими границами ответственности:

1.  **Transport (HTTP-слой, хендлеры):** Отвечает за работу с `gin.Context`, чтение параметров/тела запроса, выбор правильного HTTP-кода и формирование JSON-ответа. Обращается к сервисам только через интерфейсы.
2.  **Services (Бизнес-логика):** Содержит сценарии предметной области и валидацию данных (проверка остатков на складе, расчет скидок по промокодам, контроль переходов статусов заказа, проверка факта покупки товара перед созданием отзыва).
3.  **Repository (Работа с БД):** Отвечает исключительно за чтение и запись сущностей в PostgreSQL. Полностью изолирован от HTTP-слоя.
4.  **Models & DTO:** Структуры доменных моделей и отдельные структуры для запросов/ответов (например, `MedicineCreateRequest` и `MedicineUpdateRequest`), защищающие логику базы данных от сквозной перезаписи.

## 📦 Сущности и основные эндпоинты

### 💊 Каталог лекарств и категорий
*   `GET /medicines` — Список лекарств с фильтрацией (`?search=`, `?category_id=`, `?in_stock=`).
*   `GET /medicines/:id` — Подробная карточка лекарства с выводом динамического `avg_rating`.
*   `POST` / `PATCH` / `DELETE /medicines/:id` — Управление позициями каталога.
*   `GET /categories` | `GET /categories/:id/subcategories` — Навигация по категориям.

### 🛒 Корзина и оформление заказов
*   `GET /users/:id/cart` — Текущее состояние корзины и итоговая сумма.
*   `POST /users/:id/cart/items` — Добавление товара (с контролем остатка на складе).
*   `POST /users/:id/orders` — Создание заказа на основе корзины, ее последующая очистка и применение промокода.

### 💳 Платежи, промокоды и отзывы
*   `POST /orders/:id/payments` — Регистрация платежа. При полной оплате статус заказа автоматически переходит в `paid`.
*   `POST /promocodes/validate` — Проверка активности промокода и расчет скидки.
*   `POST /medicines/:id/reviews` — Добавление отзыва (доступно только для реально купленных пользователем лекарств).

## 🚦 Стандарт обработки ошибок

Все HTTP-ответы возвращаются в формате JSON. В случае падения валидации или бизнес-правил сервер возвращает фиксированный объект ошибки и соответствующий HTTP-статус (`400`, `404`, `422`):

```json
{
  "error": "человеко-понятное описание ошибки"
}
```

## 🚀 Запуск и разработка
1. Склонируйте репозиторий:

Bash
git clone <ссылка_на_ваш_репозиторий>

2. Настройте конфигурационный файл окружения (переменные для подключения к PostgreSQL).

3. Запустите сервер:
```
Bash
go run main.go
```
