# Бекенд для приложения онлайн-аптеки

<p>Это групповой проект, выполненный в рамках командной разработки коммерческого REST API для онлайн-аптеки на языке Go.</p>

<p><a href="https://github.com/Veoler/team-pharmacy" target="_blank">🌎 Ссылка на репозиторий проекта</a></p>

## Функционал сайта

- Управление каталогом лекарств и категорий
- Корзина пользователя с фиксацией цен в момент добавления
- Оформление заказов и автоматический контроль остатков на складе
- **Гибкая система промокодов (скидки в % и фиксированные)**
- **Поэтапная оплата заказов несколькими платежами**
- **Система отзывов и оценок к лекарствам с проверкой факта покупки**
- Четкое разделение на слои (Transport -> Service -> Repository -> Models)

## Запуск проекта

Для запуска проекта локально вам необходимо установить зависимости:

```bash
go mod tidy

```

После настройки конфигурационного файла базы данных PostgreSQL (`.env`), запустите сервер командой:

```bash
go run main.go

```

### В этом проекте я выполнял следующие задачи:

* [x] **Разработка модуля Промокодов:** создание логики валидации кодов по датам, активности и лимитам использования, а также перерасчет полей `discount_total` и `final_price` заказа.
* [x] **Разработка финансового модуля (Платежи):** реализация поддержки нескольких транзакций для одного заказа, контроль переплаты и автоматическое обновление статуса заказа на `paid` на уровне сервиса при полной оплате.
* [x] **Разработка модуля Отзывов:** написание функционала добавления/редактирования/удаления отзывов, динамический пересчет среднего рейтинга лекарства (`avg_rating`) и интеграция строгой бизнес-валидации (отзыв может оставить только тот, кто реально покупал данный товар).
* [x] Разработка соответствующих DTO-структур для безопасной записи и обновления данных по своим сущностям.
* [x] Изоляция SQL-запросов на уровне слоя `repository` для сущностей платежей, промокодов и отзывов.

---

### Для выполнения поставленных задач я использовал следующие библиотеки и подходы:

* **Gin Gonic** — для построения чистых и понятных REST-эндпоинтов, обработки HTTP-статусов и валидации JSON.
* **Интерфейсы в Go** — для связывания слоев `transport`, `services` и `repository`, что обеспечивает независимое тестирование бизнес-логики.
* **Data Transfer Objects (DTO)** — для входящих запросов на создание и частичное обновление (`PATCH`) сущностей, чтобы защитить таблицы БД от сквозной перезаписи.
